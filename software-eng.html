Chapter 7: Software Engineering

Term Definition
Software Engineering Branch of computer science that deals with the design, development, testing, and maintenance of software applications
Integrated Development Environment (IDE) Software application that helps programmers develop software code efficiently (like Visual Studio)
Association for Computing Machinery (ACM) Purpose of ACM: Advancing the art, science, engineering, and application of computing
Benefits of ACM Membership: Network of highly dedicated student and professional peers, job center, and opportunities for hands-on experience
Student ACM Membership: Career help, included coding subscriptions for magazines and sites to help students learn
Institute of Electrical and Electronics Engineers (IEEE) Purpose of IEEE: Foster technological innovation and excellence for the benefit of humanity
Benefits of IEEE Membership: Reduced conference registration rates, insurance plans, discounts on financial services and on home and office services and on travel services
Student IEEE Membership: Access to a vast network of tech and engineering professionals to collaborate, as well as resources and opportunities for hands-on experience
Software Life Cycle: Development phase Requirements Analysis: Process used to determine the needs and expectations of a new product
Design: Planning and creating the hardware and software components of a computer system
Implementation: Realization of a technical specification or algorithm as a program, software component, or other computer system through computer programming and deployment
Testing: Checking the quality, functionality, and performance of a software product before launching
Modular Implementation Approach of separating the existing features in the application module by module and making the features independent from each other
Unified Modeling Language (UML) Visual modeling language that helps software developers visualize and construct new systems
Use Case Concept used in software development, product design, and other fields to describe how a system can be used to achieve specific goals or tasks
Use Case Diagram Graphical depiction of a user's possible interactions with a system
Data Relationships: One-to-One Process of providing all students with access to electronic devices, whether they be laptops, tablets, or any other form of device that can be used for learning
Data Relationships: One-to-Many When one row in the first table is linked to many rows in the second, but only one row in the second table is linked to one row in the first
Data Relationships: Many-to-Many When multiple records in a table are associated with multiple records in another table
Design Pattern Reusable solution of how to solve a common problem when designing an application or system
Software Quality Assurance (SQA) Practice of monitoring all software engineering processes, methods, and work products to ensure compliance against defined standards
Software Testing: Glass Box Testing Testing technique that examines the program structure and derives test data from the program logic/code
Software Testing: Black Box Testing Testing that is performed with no knowledge of a system's internals
Software Testing: Beta Testing Opportunity for real users to use a product in a production environment to uncover any bugs or issues before a general release
Software Testing: Alpha Testing Initial phase of validating whether a new product will perform as expected
Documentation: User Documentation Information created to help users understand and use a product
Documentation: System Documentation Provides an overview of the system and helps engineers and stakeholders understand the underlying technology
Documentation: Technical Documentation Term that encompasses all written documents and materials dealing with software product development
Intellectual Property Law: Software License Document that provides legally binding guidelines for the use and distribution of software
Intellectual Property Law: Software Disclaimer Software developer's formal notice to the user of its software that the developer is denying any and all liability for any damages arising out of using the software as well as denying any implied warranties
Intellectual Property Law: Best Defenses Against Software Liability Claims Sound design methods, state-of-the-art quality practices, and comprehensive testing
