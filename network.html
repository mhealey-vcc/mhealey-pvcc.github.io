Network Terms and Definitions

Term Definition
Network Classifications LAN: collection of devices connected together in one physical location, such as a building, office, or home
MAN: computer network that connects computers within a metropolitan area
WAN: similar to LAN but instead connect users and applications in geographically dispersed locations
Network Models bus network: local area network topology in which each node, a workstation or other device, is connected to a main cable or link
star network:local area network (LAN) topology in which all nodes, personal computers, workstations, or other devices are directly connected to a common central computer
Protocols CSMA/CD Ethernet: media access control protocol that is used in Ethernet networks to regulate access to the shared transmission medium
CSMA/CA Wifi: protocol that helps data sending and receiving processes across a network
Combining networks repeater: network hardware device that regenerates an incoming signal from the sender before retransmitting it to the receiver
bridge: single, aggregate network from multiple communication networks or network segments
switch: connects devices in a network to each other, enabling them to talk by exchanging data packets
router: device that connects two or more packet-switched networks
gateway: network node used in telecommunications that connects two networks with different transmission protocols together
Client/Server model P2P (peer to peer): network in which two or more PCs share files and access to devices such as printers without requiring a separate server computer or server software
distributed system: any environment where multiple computers or devices are working on a variety of tasks and components, all spread across a network
cloud computing: the delivery of different services through the Internet, including data storage, servers, databases, networking, and software
Internet Addressing Tier 1: private networks that allow traffic from other Tier 1 networks to transit their backbones without a fee
Tier 2: an internet service provider which engages in the practice of peering with other networks, but which also purchases IP transit to reach some portion of the internet
Tier 3: provider that strictly purchases internet transit
IP address ICANN:
DNS: turns domain names into IP addresses, which allow browsers to get to websites and other internet resources
TLD: the last segment of a domain name, or the part that follows immediately after the "dot" symbol
Country-code TLDs:
Internet Protocols FTP: network protocol for transmitting files between computers over Transmission Control Protocol/Internet Protocol (TCP/IP) connections
HTTP: application protocol for distributed, collaborative, hypermedia information systems that allows users to communicate data on the World Wide Web
SMTP: messaging protocol governing electronic-mail transmission in Transmission Control Protocol/Internet Protocol (TCP/IP) networks
World Wide Web URL: the address of a given unique resource on the Web
client-side activities: everything in a web application that is displayed or takes place on the client (end user device)
server-side activities: everything that happens on the server, instead of on the client
Internet four-layer software application layer: the layer for collecting and managing the data that gets produced
transport layer: layer is responsible for taking the data packets and sending them to the correct computer
network layer: part of the Internet communications process where these connections occur, by sending packets of data back and forth between different networks
link layer: descriptive realm of networking protocols that operate only on the local network segment (link) that a host is connected to
Internet is packet-switching network: transfer of small pieces of data across various networks
fault tolerant: system's ability to continue operating uninterrupted despite the failure of one or more of its components
redundant: process of providing multiple paths for traffic so that data can keep flowing even in the event of a failure
Cybersecurity malware: file or code that infects, explores, steals or conducts virtually any behavior an attacker wants
virus: type of malware that spreads from computer to computer without having to drop a file-based copy of itself in any of the affected computers
worm: standalone malware computer program that replicates itself in order to spread to other computers
Trojan horse: program downloaded and installed on a computer that appears harmless but is actually malicious
spyware: malicious software that enters a user's computer, gathers data from the device and user, and sends it to third parties without their consent
phishing: practice of sending fraudulent communications that appear to come from a legitimate and reputable source
DDoS attack: cybercrime in which the attacker floods a server with internet traffic to prevent users from accessing connected online services and sites
Protection firewall: network security device that monitors and filters incoming and outgoing network traffic
spam filter: identifies emails that attackers or marketers use to send unwanted or dangerous content
proxy server: system running on a computer that acts as an intermediary between an endpoint device
antivirus software: security program designed to prevent, detect, search and remove viruses and other types of malware from computers, networks and other devices
cryptography: process of hiding or coding information so that only the person a message was intended for can read it
public key encryption: method of encrypting or signing data with two different keys and making one of the keys, the public key, available for anyone to use
